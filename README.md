# sansmax

Неофициальный консольный клиент для мессенджера MAX (`api.oneme.ru`) на основе разобранного бинарного протокола.

## Что это

sansmax подключается к `api.oneme.ru` по TLS, общается на бинарном протоколе поверх MsgPack, авторизуется по SMS на твой номер и дальше работает от твоего аккаунта. Он читает список чатов, выгружает историю переписки в JSONL и TXT, показывает профиль и контакты, отдаёт и меняет настройки облачного пароля (2FA).

Протокол разобран реверс-инжинирингом. Официального API у MAX нет, формат ответов сервера может поменяться без предупреждения — тогда часть парсеров перестанет попадать в данные.

## Дисклеймер

Проект не связан с MAX, VK и их разработчиками. Инструмент рассчитан на работу с собственным аккаунтом в учебных и личных целях. Ответственность за соблюдение правил сервиса и применимого законодательства лежит на том, кто запускает программу. Гарантий нет, код предоставляется как есть.

## Требования

- Python 3.11 или новее (используется синтаксис `str | None`).
- Пакет `msgpack`.
- Доступ к номеру телефона аккаунта — на него придёт SMS-код.

## Запуск из исходника

```
pip install msgpack
python sansmax.py
```

## Запуск .exe (Windows)

Готовый `sansmax.exe` лежит в разделе Releases репозитория. Python для него не нужен. Скачай файл и запусти из консоли (PowerShell или cmd), чтобы видеть вывод и вводить SMS-код:

```
.\sansmax.exe
```

Рядом с собой программа создаст файл токена и папку выгрузок, поэтому клади `.exe` в отдельный каталог, а не в системную папку.

## Первый вход

1. Запусти программу. Если в коде осталась заглушка номера, она спросит телефон — введи в формате `+79001234567`.
2. Введи SMS-код из сообщения.
3. Если на аккаунте включён облачный пароль (2FA), программа попросит его отдельно.
4. После успешного входа токен сохраняется в `max_login_token.txt` рядом с программой. Следующие запуски идут без SMS, пока токен жив.

Файл `max_login_token.txt` — это фактически доступ к аккаунту без пароля. Не коммить его, не пересылай, не клади в публичные места. На Linux и macOS на него автоматически ставится режим `600`.

## Меню

Чтение:

- `5` — показать `chatId`, найденные при логине.
- `1` — CHAT_INFO по `chatId`.
- `2` и `20` — история чата в консоль.
- `6` — профиль текущего аккаунта.
- `7` — CONTACT_INFO по `contactId`.
- `8` — поиск контакта по номеру телефона.

Выгрузка:

- `18` — выгрузить историю одного чата в JSONL и TXT (папка `max_exports`).
- `19` — выгрузить историю всех найденных чатов.
- `21` — TEST_HISTORY_VARIANTS, отладка пагинации истории.

Отправка:

- `3` — индикатор «печатает» в чате.
- `4` — отправить текстовое сообщение в чат.

Облачный пароль и сессии (всё это работает с твоим собственным аккаунтом):

- `9` — AUTH_2FA_DETAILS, статус облачного пароля.
- `10` — AUTH_CREATE_TRACK, отладочный перебор типов трека.
- `11` — проверить текущий 2FA-пароль.
- `12` — проверить кандидат в новый пароль.
- `13` — проверить hint.
- `14` — сменить 2FA-пароль, требует подтверждения словом `CHANGE`.
- `15` — выйти из текущей сессии (LOGOUT) и удалить токен.
- `16` — список активных сессий.
- `17` — закрыть сессию по `sessionId`.

Служебное:

- `99` — удалить сохранённый токен и форсировать повторную SMS-авторизацию.
- `0` — выход.

## Куда пишутся файлы

- `max_login_token.txt` — токен текущей сессии.
- `max_exports/` — результат выгрузок: `chat_<id>.jsonl`, `chat_<id>.txt`, сырые страницы `.bin`, `summary.json`.

## Сборка .exe самостоятельно

```
pip install pyinstaller msgpack
pyinstaller --onefile --console --name sansmax sansmax.py
```

Готовый файл появится в `dist/sansmax.exe`.

## Про номер телефона в коде

В опубликованной версии настоящий номер заменён на заглушку `+79001234567`. Править её в исходнике не обязательно: при первом входе без сохранённого токена программа спросит номер сама.
